Rendering Demo

Part of Directx 11 rendering framework demo written in C++. Showcases Atrium Sponza Palace model improved and published by Crytek that was originally created by Frank Meinl. Scene contains a textured model imported using Assimp library that contains different types of materials.

Normal and specular textures had to be authored separately by hand. Scene features a point light casting shadows that heavily utilizes instancing along with it's cube shadow mapping for major performance improvements. Shaders written in HLSL render these materials realistically with normal and specular as they are lit by the light source.

Terrain Rendering

Custom terrain generation and rendering engine written from scratch in C++ and utilizing DirectX 11. Features random terrain generation creating landscapes with a single press of a button. Terrain's vertices are created from a heightmap generated using diamond-square algorithm. Terrain's texture is created by running a shader that turns it's heightmap into a normal texture.

Main feature implements geomipmapping that splits the entire terrain into small patches, each with different resolution based on the distance from the view. Terrain patches are later carefully connected to each other in order to avoid seams. This feature along with frustum culling improves performance from 10 FPS to 150 FPS on 1024x1024 heightmap terrain. Rendering a terrain requires only a single draw call.

Custom framework also features a day/night cycle with a sky dome rendered using a third-party atmospheric scattering shader.

Deferred Rendering

Implementation of deferred renderer that was written in days of Killzone 2 and S.T.A.L.K.E.R. games that brought this technique into wide popularity. Features a renderer written in C++ and utilizing Directx 11 that easily handles huge number of light sources that cannot be drawn easily using traditional forward rendering techniques. Showcases a scene with Stanford Bunny and normal/specular mapping.

Scene is first rendered to many different layers on a GBuffer; each containing information like albedo, normal and specular. Second pass involves combining these layers and drawing lights separately. Scissor tests are used to improve the performance by only drawing into part of the framebuffer that is affected by a rendered light source.

HDR Rendering

Part of a custom Directx 11 rendering framework demo written in C++. Presents HDR feature that renders a scene into a high range framebuffer that is later mapped into a color range that can be displayed by a monitor.

Utilizes a histogram that mimics eye adaptation to light/dark environments and applies bloom effect for bright spots on the scene.

Text Rendering

Custom framework for loading fonts and drawing text strings, written in C++. Font glyphs are first rendered to a bitmap using GDI+ and then drawn with DirectX 11 API. Font cache stores previously rendered glyphs for later use. Quering text metrics and wrapping information is also implemented.
